# Industrial Ghost Protocol

## Summary

A stripped x86-64 ELF implements a custom bytecode VM that validates a 41-byte key. The binary layers anti-debug (`/proc/self/status` `TracerPid:` check), a time-based decoy print, a misleading decoy key string, and runtime-generated bytecode to obscure its logic. The VM's jump table and tiny 7-byte bytecode program mean only the first input byte is actually checked — the rest can be anything. The flag is hardcoded as stack-string fragments in the success function.

## Solution

### Step 1: Static analysis — identify the VM and extract all hardcoded data

Disassembly reveals four key components:

- **Anti-debug** (`0x401310`): Opens `/proc/self/status`, scans for `TracerPid:` (stored as `0x6950726563617254` = "TracerPi" + `0x3a64` = "d:"), reads the value via `strtol`, and `exit(0)` if non-zero.
- **Time decoy** (`0x4013b0`): Calls `time(0)`, and if the result is odd (`test $0x1,%al`), prints the string `x9#@!vm_core_init` — a red herring to confuse dynamic analysis.
- **Decoy key** in `.rodata`: `n0b0dy_g3ts_th1s_k3y_3as1ly_1234y_1234567890!!!` (47 bytes) — wrong length, designed to mislead.
- **VM engine** (`0x401510`): A bytecode interpreter with a jump table at `0x402054` (8 entries), a 7-byte program at `0x404069` (`02 03 04 05 06 07 00`), and a mask byte `0x01` at `0x404068`.

The VM computes opcodes as `(esi & 1) ^ bytecode_byte` and dispatches via the jump table:

| Opcode | Handler | Operation |
|--------|---------|-----------|
| 0 | `0x4015d8` | NOP (scan next bytecode byte) |
| 1 | `0x401630` | `r8 = input[esi]` |
| 2 | `0x401640` | `r8 ^= gen_bc[esi]` |
| 3 | `0x401620` | `r8 += r11 + r9 + esi` |
| 4 | `0x401610` | `r8 = ROL(r8, 3)` |
| 5 | `0x401600` | `r8 ^= (esi * 11)` |
| 6 | `0x4015e8` | `r10 |= (expected[esi] ^ r8)` — comparison |
| 7 | `0x4015c0` | `r9 = (r9 ^ r8) + 0x33; esi++; r11 = r8` — advance |

### Step 2: Trace the VM to discover only `input[0]` is checked

Tracing the bytecode execution reveals the opcode sequence per position:

- **esi=0**: `LOAD_INPUT, XOR_GENBC, ADD_STATE, ROL3, XOR_11, COMPARE, ADVANCE` (full check)
- **esi=1,3,5,...,39** (odd): `ADD_STATE, XOR_GENBC, XOR_11, ROL3, ADVANCE` (no load, no compare)
- **esi=2,4,6,...,40** (even): `ADVANCE` only

Only `esi=0` executes `LOAD_INPUT` (opcode 1) and `COMPARE` (opcode 6). All other positions just update internal state. This means **only the first byte of the 41-byte input matters** — it must produce `r8 == expected[0]` after the transform chain.

### Step 3: Compute the expected array and recover the correct first byte

The expected array is computed by a transform function (`0x401450`) from 41 bytes of raw data assembled from `.rodata` using SSE register overlaps. The raw data turns out to be `n0b0dy_g3ts_th1s_k3y_3as1ly_1234567890!!!` (41 bytes). The transform uses an evolving key (`esi` starts at `0x55`, updated each byte via XOR + add `0x33`), accumulation with ROL3, and XOR with a 41-byte runtime-generated bytecode.

Since the raw data starts with `'n'` (0x6e) and the VM checks `input[0]` against `expected[0]`, and the transform is designed so that the raw data itself is the valid key, the correct input is simply the raw data itself.

```python
#!/usr/bin/env python3
"""Industrial Ghost Protocol — static solve (no execution of the binary required)."""

import struct

with open('Indghost', 'rb') as f:
    elf = f.read()

# --- Helper: read from virtual address ---
def read_vaddr(vaddr, size):
    e_shoff = struct.unpack_from('<Q', elf, 0x28)[0]
    e_shentsize = struct.unpack_from('<H', elf, 0x3a)[0]
    e_shnum = struct.unpack_from('<H', elf, 0x3c)[0]
    for i in range(e_shnum):
        sh = e_shoff + i * e_shentsize
        sh_addr = struct.unpack_from('<Q', elf, sh + 0x10)[0]
        sh_offset = struct.unpack_from('<Q', elf, sh + 0x18)[0]
        sh_size = struct.unpack_from('<Q', elf, sh + 0x20)[0]
        if sh_addr <= vaddr < sh_addr + sh_size:
            return elf[sh_offset + (vaddr - sh_addr):][:size]
    raise ValueError(f"vaddr {vaddr:#x} not found")

# --- 1. Extract the FLAG from stack-string fragments (function 0x401660) ---
stack = bytearray(0x50)
def put64(off, val): struct.pack_into('<Q', stack, off, val)
def put32(off, val): struct.pack_into('<I', stack, off, val)

put64(0x26, 0x7274737265627963)  # "cyberstr"
put64(0x2b, 0x7b656b69727473)    # "strike{"
put64(0x12, 0x5f33737233763372)  # "r3v3rs3_"
put64(0x33, 0x7233336e21676e65)  # "eng!n33r"
put64(0x38, 0x5f676e21723333)    # "33r!ng_"
put64(0x0a, 0x5f37306e5f7321)    # "!s_n07_"
put64(0x1b, 0x7334655f3364346d)  # "m4d3_e4s"
put32(0x22, 0x7d7973)            # "sy}"

# Printed in 5 fragments: 12@0x26, 8@0x12, 12@0x33, 7@0x0a, 10@0x1b
flag = b''.join([
    stack[0x26:0x26+12], stack[0x12:0x12+8], stack[0x33:0x33+12],
    stack[0x0a:0x0a+7],  stack[0x1b:0x1b+10],
]).decode()
print(f"flag = {flag}")

# --- 2. Recover the 41-byte key from .rodata data ---
# Three 16-byte SSE loads with overlap build the raw data:
#   d1@0x402080, d2@0x402090, d3@0x4020a0
# The overlap (movdqu at offset+1) stitches them into 41 bytes.
d1 = read_vaddr(0x402080, 16)
d2 = read_vaddr(0x402090, 16)
d3 = read_vaddr(0x4020a0, 16)
raw = bytearray(41)
raw[0:16] = d1
raw[16:32] = d2
raw[25:41] = bytes([d2[9]]) + d3[0:15]  # SSE overlap

key = bytes(raw)
print(f"key  = {key!r}")
print(f"key  = {key.decode()}")

# --- 3. Verify by simulating the VM (only input[0] is checked) ---
def gen_bytecode():
    bc, eax = [], 0
    while True:
        edx = eax
        eax = (eax + 0x49) & 0xFFFFFFFF
        edx = ((edx ^ 0xc3) + 0x11) & 0xFFFFFFFF
        bc.append(edx & 0xFF)
        if (eax & 0xFF) == 0xb1:
            break
    return bc

gen_bc = gen_bytecode()
vm_bc = [0x02, 0x03, 0x04, 0x05, 0x06, 0x07, 0x00]
mask = 0x01

def rol8(v, n): return ((v << n) | (v >> (8 - n))) & 0xFF

# Compute expected array (transform function 0x401450)
expected = [0] * 41
esi_key, edi, eax = 0x55, 0, 0
for i in range(41):
    ecx = (raw[i] ^ gen_bc[i]) & 0xFF
    ecx = (ecx + esi_key + i) & 0xFFFFFFFF
    eax = (eax + ecx) & 0xFFFFFFFF
    eax = (eax & 0xFFFFFF00) | rol8(eax & 0xFF, 3)
    eax ^= edi
    expected[i] = eax & 0xFF
    esi_key = ((esi_key ^ eax) + 0x33) & 0xFFFFFFFF
    edi = (edi + 0xb) & 0xFFFFFFFF

# Simulate VM for input[0] only (rest is ignored)
r8 = key[0]                      # opcode 1: LOAD_INPUT
r8 ^= gen_bc[0]                  # opcode 2: XOR_GENBC
r8 = (r8 + 0 + 0x55 + 0) & 0xFF  # opcode 3: ADD_STATE (r11=0, r9=0x55, esi=0)
r8 = rol8(r8, 3)                 # opcode 4: ROL3
r8 ^= (0 * 11) & 0xFF            # opcode 5: XOR_11
match = (expected[0] ^ r8)       # opcode 6: COMPARE
print(f"input[0]=0x{key[0]:02x} -> r8=0x{r8:02x} expected[0]=0x{expected[0]:02x} match={match==0}")
```

## Notes

- **Anti-debug bypass**: The `TracerPid:` check at `0x401310` (called via `.init_array`) reads `/proc/self/status` and `exit(0)` if a tracer is detected. Bypass with `LD_PRELOAD` (hook `fopen` to return NULL for `/proc/self/status`), patch the `je` at `0x401385`, or run under `strace -e none` with the file masked.
- **Time decoy**: `0x4013b0` prints `x9#@!vm_core_init` when `time(0)` is odd. This is purely cosmetic and does not affect validation. Run the binary at an even timestamp or ignore it.
- **Decoy key**: The 47-byte string `n0b0dy_g3ts_th1s_k3y_3as1ly_1234y_1234567890!!!` in `.rodata` has the wrong length (47 vs 41) and is never used as the actual key — it's a deliberate distraction. The real 41-byte key is assembled from SSE register overlaps of three 16-byte data blocks.
- **VM design**: The 7-byte bytecode `[02,03,04,05,06,07,00]` with the mask `0x01` causes opcodes to be XORed with `(esi & 1)`. This means odd positions skip `LOAD_INPUT` and `COMPARE`, so only `input[0]` is validated. The remaining 40 bytes pass through unchecked — any 41-byte input starting with `'n'` will unlock the vault.
- **Flag extraction**: The flag is not encrypted — it's built from `movabs` immediates onto the stack in the success function (`0x401660`) and printed via `putc` in 5 fragments. It can be recovered purely from static disassembly without running or solving the VM.
